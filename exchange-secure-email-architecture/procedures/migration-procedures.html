🔒 Exchange Secure Email Migration

Implementation Procedures for AVI, Exchange, and Applications

From Legacy Port 25 to SMTPS/EWS

Migration Overview

ℹ️ Purpose: This guide provides step-by-step procedures for migrating from legacy plaintext SMTP (Port 25) to secure protocols (SMTPS and EWS) for Exchange Server SE.

Migration Phases

1

Coexistence

Duration: 30-60 days

Deploy secure connectors while maintaining legacy Port 25 for backward compatibility.

2

Migration

Duration: 60-90 days

Migrate applications to SMTPS or EWS based on requirements.

3

Decommission

Duration: 30 days validation

Disable Port 25 after confirming zero traffic for 30+ days.

Decision Matrix

Question Protocol Key Requirements
Does the application only send emails? SMTPS (Port 587/465) TLS 1.2+, No mailbox needed, L4 passthrough
Does the application need to move/delete emails? EWS (HTTPS) Dedicated mailbox, Modern Auth, L7 bridging

🔀 AVI Load Balancer Configuration

Procedure 1: SMTPS L4 Passthrough (Ports 587/465)

🎯 Goal: Configure Layer 4 passthrough to preserve Extended Protection while using SNAT for IP trust simplification.
1 Create Health Monitor (TCP)
  • Navigate to Templates → Health Monitors
  • Click Create
  • Name: HM-Exchange-SMTP-TCP
  • Type: TCP
  • Send Interval: 10 seconds
  • Successful Checks: 2, Failed Checks: 3
2 Create Backend Pool
  • Navigate to Applications → Pools
  • Name: Pool-Exchange-SMTPS
  • Default Server Port: 587
  • Add all Exchange CAS server IPs
  • Health Monitor: HM-Exchange-SMTP-TCP
3 Create Virtual Service - Port 587 (STARTTLS)
  • Navigate to Applications → Virtual Services
  • Name: VS-Exchange-SMTP-587
  • Application Type: L4
  • VIP: <Your Load Balancer VIP>
  • Port: 587
  • Default Pool: Pool-Exchange-SMTPS
  • ⚠️ Enable SNAT → Auto Allocate
4 Create Virtual Service - Port 465 (Implicit TLS)
  • Same as Port 587, but:
  • Name: VS-Exchange-SMTP-465
  • Port: 465
  • Default Server Port: 465
5 Verify Configuration 
# Test STARTTLS on Port 587
openssl s_client -connect <VIP>:587 -starttls smtp

# Test Implicit TLS on Port 465
openssl s_client -connect <VIP>:465
6 Document AVI SE Subnet
  • Navigate to Infrastructure → Service Engines
  • Note the Data Network IPs (e.g., 10.10.50.10, 10.10.50.11)
  • Identify subnet (e.g., 10.10.50.0/24)
  • 📝 Provide this subnet to Exchange team for Receive Connector configuration

Procedure 2: EWS L7 Bridging (HTTPS)

⚠️ Important: Certificate on AVI must match Exchange certificate for Extended Protection to work.
1 Import SSL Certificate
  • Navigate to Templates → Security → SSL/TLS Certificates
  • Import certificate matching Exchange server certificate
  • Ensure Common Name or SAN matches Exchange external URL
2 Create Health Monitor (HTTPS)
  • Name: HM-Exchange-EWS
  • Type: HTTPS
  • Path: /ews/healthcheck.htm
  • Method: GET
  • Response Code: 2xx, 3xx
3 Create Persistence Profile
  • Name: Persistence-Exchange-EWS
  • Type: HTTP Cookie
  • Timeout: 20 minutes
4 Create Backend Pool
  • Name: Pool-Exchange-EWS
  • Default Server Port: 443
  • Enable SSL to backend
  • Add all Exchange CAS server IPs
  • Health Monitor: HM-Exchange-EWS
5 Create Virtual Service
  • Name: VS-Exchange-EWS
  • Application Type: HTTP/HTTPS
  • Port: 443
  • SSL Certificate: Select imported certificate
  • Default Pool: Pool-Exchange-EWS
  • Persistence Profile: Persistence-Exchange-EWS
6 Verify Configuration 
# Test EWS health check
curl -k https://<VIP>/ews/healthcheck.htm

# Expected: 200 OK response

📧 Exchange Server Configuration

Procedure 1: Create SMTPS Receive Connector

📝 Prerequisites: AVI Service Engine subnet documented from AVI configuration
1 Create Receive Connector - Port 587 
# Exchange Management Shell (PowerShell)
$CASServer = $env:COMPUTERNAME
$AviSubnet = "10.10.50.0/24"  # Replace with your AVI SE subnet

New-ReceiveConnector `
    -Name "Secure Relay - Port 587" `
    -Server $CASServer `
    -TransportRole FrontendTransport `
    -Bindings "0.0.0.0:587" `
    -RemoteIPRanges $AviSubnet `
    -AuthMechanism Tls `
    -PermissionGroups AnonymousUsers `
    -RequireTLS $true
2 Configure Connector Settings 
$Connector = Get-ReceiveConnector "$CASServer\Secure Relay - Port 587"

Set-ReceiveConnector $Connector -Identity $Connector.Identity `
    -RequireTLS $true `
    -ExtendedProtectionPolicy Require `
    -MaxMessageSize 35MB `
    -MessageRateLimit Unlimited
3 Grant Relay Permissions 
Get-ReceiveConnector "$CASServer\Secure Relay - Port 587" | 
    Add-ADPermission -User "NT AUTHORITY\ANONYMOUS LOGON" `
    -ExtendedRights "ms-Exch-SMTP-Submit", "ms-Exch-SMTP-Accept-Any-Recipient"
4 Create Connector for Port 465 (Optional) 
New-ReceiveConnector `
    -Name "Secure Relay - Port 465" `
    -Server $CASServer `
    -TransportRole FrontendTransport `
    -Bindings "0.0.0.0:465" `
    -RemoteIPRanges $AviSubnet `
    -AuthMechanism Tls `
    -PermissionGroups AnonymousUsers `
    -RequireTLS $true
5 Verify Configuration 
Get-ReceiveConnector | Where-Object {$_.Name -like "*Secure Relay*"} | 
    Select Name, Server, Bindings, RemoteIPRanges, RequireTLS | Format-Table

Procedure 2: Configure EWS Virtual Directory

1 Configure EWS for Modern Authentication 
$CASServers = @("EXCH-CAS01", "EXCH-CAS02", "EXCH-CAS03")
$ExternalURL = "https://mail.contoso.com"

foreach ($Server in $CASServers) {
    $EWS = Get-WebServicesVirtualDirectory -Server $Server
    
    Set-WebServicesVirtualDirectory -Identity $EWS.Identity `
        -InternalUrl "$ExternalURL/ews/Exchange.asmx" `
        -ExternalUrl "$ExternalURL/ews/Exchange.asmx" `
        -BasicAuthentication $false `
        -WindowsAuthentication $true `
        -OAuthAuthentication $true `
        -ExtendedProtectionTokenCheck Require
}
2 Create Service Account Mailbox 
New-Mailbox -UserPrincipalName ewsprocessing@contoso.com `
    -Alias ewsprocessing `
    -Name "EWS Processing Service Account" `
    -Password (ConvertTo-SecureString -String "ComplexP@ssw0rd!" -AsPlainText -Force) `
    -ResetPasswordOnNextLogon $false

Set-Mailbox -Identity ewsprocessing@contoso.com `
    -HiddenFromAddressListsEnabled $true
3 Grant Application Impersonation (if needed) 
New-ManagementRoleAssignment -Name "EWS-Application-Impersonation" `
    -Role ApplicationImpersonation `
    -User ewsprocessing@contoso.com
4 Verify EWS Configuration 
Get-WebServicesVirtualDirectory | 
    Select Server, InternalUrl, OAuthAuthentication, ExtendedProtectionTokenCheck | 
    Format-Table

Procedure 3: Monitor Port 25 Usage (Phase 1)

ℹ️ Coexistence Phase: Since Port 25 already only allows load balancer IPs, no restriction needed. Monitor usage to track migration progress.
1 Document Current Port 25 Configuration 
$LegacyConnector = Get-ReceiveConnector | 
    Where-Object {$_.Bindings -like "*:25"} | 
    Select -First 1

# Document current settings
$LegacyConnector | Select Name, Bindings, RemoteIPRanges, PermissionGroups | Format-List

# Backup current settings
$LegacyConnector | Export-Clixml -Path "C:\Temp\LegacyConnector-Backup.xml"
2 Monitor Port 25 Traffic 
# Check message tracking for Port 25 usage
Get-MessageTrackingLog -Start (Get-Date).AddDays(-7) -ResultSize Unlimited | 
    Where-Object {$_.Source -eq "SMTP" -and $_.ConnectorId -like "*$($LegacyConnector.Name)*"} | 
    Group-Object -Property ClientHostname | 
    Select Count, Name | 
    Sort Count -Descending | 
    Export-Csv "C:\Temp\Port25-Usage.csv" -NoTypeInformation

Write-Host "Port 25 traffic logged to C:\Temp\Port25-Usage.csv"
3 Set Up Weekly Monitoring

Create a scheduled task to run the monitoring script weekly and track the decrease in Port 25 traffic as applications are migrated to SMTPS/EWS.

💻 Application Migration

Migration Path 1: SMTPS Port 587 (STARTTLS)

✓ Use Case: Applications that only send email notifications (no mailbox access)

Example 1: PowerShell Script

$SMTPServer = "<AVI-VIP>"
$SMTPPort = 587

Send-MailMessage -From "sender@contoso.com" `
    -To "recipient@example.com" `
    -Subject "Test Subject" `
    -Body "Test Body" `
    -SmtpServer $SMTPServer `
    -Port $SMTPPort `
    -UseSsl

Example 2: Python Script

import smtplib
from email.mime.text import MIMEText

smtp_server = "<AVI-VIP>"
smtp_port = 587

msg = MIMEText("Test email body")
msg['Subject'] = "Test Subject"
msg['From'] = "sender@contoso.com"
msg['To'] = "recipient@example.com"

with smtplib.SMTP(smtp_server, smtp_port) as server:
    server.starttls()
    server.send_message(msg)

Example 3: .NET Application

using System.Net.Mail;

SmtpClient client = new SmtpClient("<AVI-VIP>", 587);
client.EnableSsl = true;
client.DeliveryMethod = SmtpDeliveryMethod.Network;
client.UseDefaultCredentials = false;
client.Credentials = null;

MailMessage message = new MailMessage(
    "sender@contoso.com",
    "recipient@example.com",
    "Test Subject",
    "Test Body"
);

client.Send(message);

Example 4: Linux Postfix

# Edit /etc/postfix/main.cf
relayhost = [<AVI-VIP>]:587
smtp_tls_security_level = encrypt
smtp_use_tls = yes

# Restart Postfix
systemctl restart postfix

# Test
echo "Test email" | mail -s "Test Subject" user@example.com

Migration Path 2: EWS Mailbox Processing

✓ Use Case: Applications that need to read, move, or delete emails from a mailbox

Example 1: PowerShell Script

# Load EWS Managed API
Add-Type -Path "Microsoft.Exchange.WebServices.dll"

$service = New-Object Microsoft.Exchange.WebServices.Data.ExchangeService
$service.Url = "https://<AVI-VIP>/ews/Exchange.asmx"
$service.Credentials = New-Object System.Net.NetworkCredential(
    "ewsprocessing@contoso.com", "password"
)

# Get Inbox
$inbox = [Microsoft.Exchange.WebServices.Data.Folder]::Bind(
    $service, 
    [Microsoft.Exchange.WebServices.Data.WellKnownFolderName]::Inbox
)

# Find items
$itemView = New-Object Microsoft.Exchange.WebServices.Data.ItemView(100)
$findResults = $service.FindItems(
    [Microsoft.Exchange.WebServices.Data.WellKnownFolderName]::Inbox, 
    $itemView
)

# Process each item
foreach ($item in $findResults.Items) {
    Write-Host "Processing: $($item.Subject)"
    
    # Move to Processed folder
    $item.Move($processedFolderId) | Out-Null
}

Example 2: Python Script

from exchangelib import Credentials, Account, Configuration

credentials = Credentials(
    username='ewsprocessing@contoso.com', 
    password='password'
)

config = Configuration(
    server='<AVI-VIP>', 
    credentials=credentials
)

account = Account(
    primary_smtp_address='ewsprocessing@contoso.com',
    config=config,
    autodiscover=False
)

# Get or create Processed folder
processed_folder = account.inbox / 'Processed'

# Process inbox items
for item in account.inbox.all():
    print(f"Processing: {item.subject}")
    
    # Move to Processed folder
    item.move(to_folder=processed_folder)

Example 3: C# Application

using Microsoft.Exchange.WebServices.Data;

ExchangeService service = new ExchangeService();
service.Url = new Uri("https://<AVI-VIP>/ews/Exchange.asmx");
service.Credentials = new NetworkCredential(
    "ewsprocessing@contoso.com", "password"
);

Folder inbox = Folder.Bind(service, WellKnownFolderName.Inbox);

ItemView view = new ItemView(100);
FindItemsResults<Item> findResults = service.FindItems(
    WellKnownFolderName.Inbox, view
);

foreach (Item item in findResults.Items)
{
    EmailMessage email = EmailMessage.Bind(service, item.Id);
    Console.WriteLine($"Processing: {email.Subject}");
    
    // Move to Processed folder
    email.Move(targetFolderId);
}
